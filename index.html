🎵 Audio Capture Options

Comparison of system audio vs tab audio capture approaches.

System Audio
Captures all system audio output
Uses systemAudio: "include". Only available when capturing a monitor. Not supported in all browsers. User must check the "Share system audio" checkbox.
Tab Audio
Captures audio from a specific browser tab
Uses audio: true in constraints. Only available when capturing a browser tab surface. Reliable across Chromium browsers.
suppressLocalAudioPlayback
Prevents echo during tab capture
When capturing a tab's audio, setting this to true mutes the tab's audio locally while still capturing it. Useful for recording or streaming.
No Audio
Video-only capture
Set audio: false for silent captures. Common for screenshots, screen recording where audio isn't needed, or privacy-sensitive contexts.

The Screen Capture API (getDisplayMedia()) is distinct from getUserMedia(). It always requires explicit user permission through a system picker dialog and cannot be started without a user gesture. Some constraints like displaySurface, selfBrowserSurface, and surfaceSwitching are Chrome-specific.
